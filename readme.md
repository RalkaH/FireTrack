# 🔥 FireTrack — Система отметки проверки средств пожаротушения

![Версия](https://img.shields.io/badge/Версия-1.1.0-blue)
![FastAPI](https://img.shields.io/badge/FastAPI-0.100+-009688?logo=fastapi)
![Python](https://img.shields.io/badge/Python-3.9+-3776AB?logo=python)
![SQLite](https://img.shields.io/badge/SQLite-Database-003B57?logo=sqlite)
![Docker](https://img.shields.io/badge/Docker-Enabled-2496ED?logo=docker)

**FireTrack** — это автоматизированная система учета и контроля состояния первичных средств пожаротушения (огнетушителей) на промышленных, складских и административных объектах.

Проект предназначен для замены бумажных журналов проверок, автоматизации контроля технического обслуживания и формирования цифровой отчетности для внутренних проверок и надзорных органов.

Система помогает:

- минимизировать человеческий фактор;
- централизованно хранить данные об оборудовании;
- отслеживать сроки проверок и перезарядки;
- формировать отчеты в электронном виде;
- контролировать состояние противопожарного инвентаря;
- соответствовать требованиям ГОСТ Р 51057-2001 и ФЗ №123.

# 🚀 Основные возможности

## 📋 Учет огнетушителей

Система позволяет регистрировать и хранить информацию о каждом огнетушителе.

## 📝 Электронный журнал проверок

Каждая проверка фиксируется в электронном журнале с сохранением метрик состояния оборудования.

## ⏰ Контроль сроков обслуживания

Автоматический расчет сроков ТО и отслеживание просроченных проверок.

## 📊 Дашборд и аналитика

Главная панель отображает общее состояние системы в реальном времени.

## 📥 Импорт и экспорт

Поддержка CSV и XLSX файлов.

# 🔐 Ролевая модель

В системе предусмотрено 3 роли доступа (RBAC).

| Роль | Возможности |
|---|---|
| admin | Полный доступ к системе и управлению пользователями |
| engineer | Доступ к проведению проверок и работе с оборудованием |
| viewer | Доступ только на чтение для просмотра аналитики |

# 📥 Установка и запуск (Локально)

## Шаг 1. Клонирование репозитория

```bash
git clone https://github.com/RalkaH/FireTrack.git
cd FireTrack
```

---

## Шаг 2. Создание виртуального окружения и установка зависимостей

### Windows

```bash
python -m venv venv
venv\Scripts\activate
```

### Linux/macOS

```bash
python3 -m venv venv
source venv/bin/activate
```

Установка зависимостей:

```bash
pip install -r requirements.txt
```

---

## Шаг 3. Запуск приложения

```bash
uvicorn main:app --reload
```

---

# 📦 Сборка проекта (Setuptools)

Проект настроен для установки как полноценный Python-пакет с помощью setuptools.

Для сборки и локальной установки пакета в режиме разработки:

```bash
pip install -e .
```

После установки приложение можно запустить через консольную команду, прописанную в setup.py:

```bash
firetrack-run
```

# 📚 Генерация документации (Sphinx)

Код проекта документирован с помощью Docstrings.

Для сборки HTML-документации:

Установка Sphinx:

```bash
pip install sphinx sphinx_rtd_theme
```

Переход в директорию документации:

```bash
cd docs
```

Сборка документации:

```bash
make html
```

Готовая документация будет доступна по пути:

```text
docs/_build/html/index.html
```

# 🐳 Развертывание через Docker

Для удобного развертывания проекта в облаке или на сервере подготовлены:

- Dockerfile
- docker-compose.yml

Все параметры конфигурации, включая подключение к БД, передаются через переменные окружения.

Запуск контейнеров:

```bash
docker-compose up -d --build
```

После запуска приложение будет доступно по адресу:

```text
http://localhost:8000
```
