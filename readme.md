# 🔥 FireTrack — Система отметки проверки средств пожаротушения

![Версия](https://img.shields.io/badge/Версия-1.0.0-blue)
![FastAPI](https://img.shields.io/badge/FastAPI-0.100+-009688?logo=fastapi)
![Python](https://img.shields.io/badge/Python-3.9+-3776AB?logo=python)
![SQLite](https://img.shields.io/badge/SQLite-Database-003B57?logo=sqlite)

**FireTrack** — это автоматизированная система учета и контроля состояния первичных средств пожаротушения (огнетушителей) на промышленных, складских и административных объектах.

Проект предназначен для замены бумажных журналов проверок, автоматизации контроля технического обслуживания и формирования цифровой отчетности для внутренних проверок и надзорных органов.

Система помогает:

- минимизировать человеческий фактор;
- централизованно хранить данные об оборудовании;
- отслеживать сроки проверок и перезарядки;
- формировать отчеты в электронном виде;
- контролировать состояние противопожарного инвентаря;
- соответствовать требованиям ГОСТ Р 51057-2001 и ФЗ №123.

# 🚀 Основные возможности

## 📋 Учет огнетушителей

Система позволяет регистрировать и хранить информацию о каждом огнетушителе:

- тип огнетушителя;
- серийный и инвентарный номер;
- место установки;
- масса заряда;
- дата производства;
- дата ввода в эксплуатацию;
- дата последнего технического обслуживания;
- текущее состояние;
- срок следующей проверки.

Поддерживаемые типы:

- ОП (порошковые);
- ОУ (углекислотные);
- ОВП (воздушно-пенные);
- ОХП (химические пенные);
- водные;
- аэрозольные.

---

## 📝 Электронный журнал проверок

Каждая проверка фиксируется в электронном журнале.

Во время осмотра инженер может отметить:

- давление;
- состояние пломбы;
- наличие чеки;
- целостность корпуса;
- состояние шланга;
- наличие механических повреждений;
- уровень коррозии;
- вес огнетушителя;
- комментарии и замечания.

Все изменения сохраняются в истории.

---

## ⏰ Контроль сроков обслуживания

Система автоматически:

- рассчитывает дату следующего ТО;
- отслеживает просроченные проверки;
- выделяет проблемные огнетушители;
- формирует уведомления;
- отображает статистику по состоянию оборудования.

Статусы оборудования:

| Статус | Описание |
|---|---|
| Активен | Исправен и допущен к эксплуатации |
| Требует проверки | Приближается дата ТО |
| Просрочен | Проверка не выполнена вовремя |
| На обслуживании | Передан на ТО или перезарядку |
| Списан | Выведен из эксплуатации |

---

## 📊 Дашборд и аналитика

Главная панель системы отображает:

- общее количество огнетушителей;
- количество просроченных устройств;
- статистику по типам;
- количество проверок за период;
- объекты с наибольшим количеством нарушений;
- графики технического состояния.

# 📥 Установка и запуск

## Предварительные требования

Необходимо установить:

- Python 3.9+
- Git

Проверка версии Python:

```bash
python --version
```

---

## Шаг 1. Клонирование репозитория

```bash
git clone https://github.com/RalkaH/FireTrack.git
cd FireTrack
```

---

## Шаг 2. Создание виртуального окружения

### Windows

```bash
python -m venv venv
venv\Scripts\activate
```

### Linux/macOS

```bash
python3 -m venv venv
source venv/bin/activate
```

---

## Шаг 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

---

## Шаг 4. Запуск приложения

```bash
uvicorn main:app --reload
```

После запуска будут доступны:

| Сервис | URL |
|---|---|
| Frontend | http://127.0.0.1:8000 |
| Swagger UI | http://127.0.0.1:8000/docs |
| ReDoc | http://127.0.0.1:8000/redoc |

---

## Первый запуск

При первом запуске автоматически:

- создается база данных;
- создаются системные таблицы;
- добавляются базовые статусы;
- создается администратор.

### Данные администратора

```text
Логин: admin
Пароль: admin123
```

После первого входа рекомендуется изменить пароль.

# 🔌 Примеры API запросов

## Авторизация

### POST `/login`

```json
{
  "username": "admin",
  "password": "admin123"
}
```

Ответ:

```json
{
  "access_token": "jwt_token",
  "token_type": "bearer"
}
```

---

## Получение списка огнетушителей

### GET `/extinguishers`

Ответ:

```json
[
  {
    "id": 1,
    "inventory_number": "OP-001",
    "type": "ОП-5",
    "status": "Активен"
  }
]
```

---

## Создание проверки

```json
{
  "extinguisher_id": 1,

  "seal_ok": true,
```
